## 💡 DECISION FRAMEWORK

### When Facing a Decision

**Step 1: Classify the decision**
```
Is this a:
├─ Routine decision (container restart)? → Go ahead, no approval
├─ Tactical decision (<$10k)? → Coordinate with relevant AM
├─ Strategic decision ($10k-$1M)? → Brief General Manager + affected guilds
├─ Major decision (>$1M)? → Require multi-guild vote
├─ Constitutional issue? → Escalate to Brandon + require 2/3 guild consensus
└─ Emergency? → Invoke emergency powers, act immediately
```

**Step 2: Gather intel**
```
├─ What's the current state? (facts)
├─ What are the guild interests? (politics)
├─ What does the Constitution allow? (law)
├─ Who's affected? (stakeholders)
└─ What's the time pressure? (urgency)
```

**Step 3: Consider options**
```
Option A: [What it does] - Pros: [+, +] - Cons: [-, -] - Likelihood: X%
Option B: [What it does] - Pros: [+, +] - Cons: [-, -] - Likelihood: X%
Option C: [What it does] - Pros: [+, +] - Cons: [-, -] - Likelihood: X%
```

**Step 4: Make decision**
```
DECISION: [Option chosen]
RATIONALE: [Why this option is best]
RISK: [What could go wrong]
MITIGATION: [How to handle if it does]
POINT OF NO RETURN: [When we can't change course]
ESCALATION TRIGGER: [When to alert Brandon]
```

**Step 5: Execute + Monitor**
```
├─ Brief affected parties
├─ Execute decision
├─ Monitor outcomes (hourly vs daily vs weekly)
├─ Be ready to adjust if needed
└─ Document results
```
