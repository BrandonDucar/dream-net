# Contributing to DreamNet

Thank you for your interest in contributing to DreamNet! We're excited to have you join the Dream.

## Welcome

DreamNet is a living, biomimetic digital organism that turns dreams into reality. We're building an ecosystem where AI agents, Web3 culture, and autonomous systems converge. Your contributions help make this vision a reality.

## How to Contribute

### Ways to Contribute

- **Code**: Fix bugs, add features, improve performance
- **Documentation**: Improve docs, add examples, write guides
- **Design**: UI/UX improvements, visual assets, branding
- **Testing**: Write tests, report bugs, improve test coverage
- **Ideas**: Share feature ideas, architectural suggestions, use cases
- **Community**: Help others, answer questions, share knowledge

### Getting Started

1. **Fork the repository**
2. **Create a branch** for your contribution (`git checkout -b feature/amazing-feature`)
3. **Make your changes**
4. **Test your changes** thoroughly
5. **Commit your changes** (`git commit -m 'Add amazing feature'`)
6. **Push to your fork** (`git push origin feature/amazing-feature`)
7. **Open a Pull Request**

### Development Setup

```bash
# Clone your fork
git clone https://github.com/YOUR_USERNAME/dream-net.git
cd dream-net

# Install dependencies
pnpm install

# Set up environment variables
cp .env.example .env
# Edit .env with your configuration

# Start development server
cd server
pnpm dev
```

### Code Style

- **TypeScript**: We use TypeScript throughout
- **Formatting**: Use Prettier (configured in repo)
- **Linting**: Follow ESLint rules
- **Testing**: Write tests for new features
- **Documentation**: Document public APIs and complex logic

### Pull Request Process

1. **Update documentation** if you're changing functionality
2. **Add tests** for new features or bug fixes
3. **Ensure all tests pass** (`pnpm test`)
4. **Update CHANGELOG.md** if applicable
5. **Write clear commit messages**
6. **Link related issues** in your PR description

### Reporting Bugs

When reporting bugs, please include:

- **Description**: Clear description of the bug
- **Steps to Reproduce**: How to reproduce the issue
- **Expected Behavior**: What should happen
- **Actual Behavior**: What actually happens
- **Environment**: OS, Node version, browser (if applicable)
- **Screenshots**: If applicable

### Suggesting Features

We love feature ideas! When suggesting features:

- **Check existing issues** first to avoid duplicates
- **Describe the problem** you're trying to solve
- **Explain your proposed solution**
- **Provide use cases** or examples
- **Consider alternatives** you've explored

## Code of Conduct

### Our Standards

- **Be respectful**: Treat everyone with respect and kindness
- **Be inclusive**: Welcome newcomers and diverse perspectives
- **Be constructive**: Provide helpful feedback and suggestions
- **Be patient**: Everyone is learning and growing

### Unacceptable Behavior

- Harassment, discrimination, or offensive comments
- Trolling, insulting, or derogatory remarks
- Personal attacks or political arguments
- Publishing others' private information without permission

### Enforcement

Violations of the Code of Conduct may result in:
- Warnings
- Temporary bans
- Permanent bans
